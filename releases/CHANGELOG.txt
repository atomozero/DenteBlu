DenteBlu — Release History

r007  2026-03-15  Profili audio + fix driver Intel + install robusto
  - A2DP Source: streaming audio SBC verso cuffie BT (SbcEncoder + A2dpSource)
  - AVRCP Target: comandi tasti cuffie (play/pause/stop/volume)
  - HFP Audio Gateway: ruolo "telefono" per cuffie HF (AT responder)
  - SCO Audio: trasporto voce bidirezionale (ScoSocket + HCI sync commands)
  - bluetooth_audio media add-on (BBufferConsumer per Media Kit)
  - SDP records: A2DP Source (0x110A), AVRCP Target (0x110C), HFP AG (0x111F)
  - Fix driver Intel h2generic: device bootloader falliti non vengono piu'
    registrati — elimina loop infinito retry/device_table_full nel syslog
  - Fix firmware naming per Intel legacy (hw_variant >= 0x11):
    formato corretto a 3 componenti ibt-<hw_variant>-<hw_revision>-<fw_revision>.sfi
    (es. ibt-19-0-4.sfi per AX201 Tiger Lake/HarrisonPeak)
  - install.sh completamente riscritto:
    * --check: diagnostica completa (hardware, firmware, syslog errors)
    * --firmware-only: scarica solo firmware Intel
    * Auto-download firmware Intel durante installazione
    * Mappa USB product ID per auto-detect senza syslog
    * Validazione dimensione file scaricati (no pagine 404)
  - install.sh: fix backup .bak dei driver — Haiku devfs carica qualsiasi
    file in drivers/bin/, quindi h2generic.bak veniva caricato come secondo
    driver con retry counter separato. Ora usa .prev e rimuove vecchi .bak
  - download_intel_fw.sh: fix `local` fuori funzione, flag --auto,
    validazione file, fallback USB product ID, detect formato 3-componenti
  - Nuovi test: bt_sco_test, bt_a2dp_source_test, bt_avrcp_test, bt_hfp_ag_test
