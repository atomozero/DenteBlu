DenteBlu — Roadmap Supporto Cuffie Bluetooth
=============================================

Obiettivo: collegare cuffie BT classiche (A2DP + HFP) a Haiku OS.
Stato attuale: lo stack ha i ruoli INVERSI (A2DP Sink + HFP HF).
Servono: A2DP Source + HFP AG + AVRCP Target + SCO audio.

Stima totale: ~4750 linee di codice, 7 fasi.


========================================================================
FASE A — SBC Encoder (~600 righe)
========================================================================

Controparte di SbcDecoder.cpp (610 righe). Codifica PCM int16 in frame
SBC come richiesto dalla specifica A2DP Appendice B.

Task A.1: Struttura base SbcEncoder
  - SbcEncoder.h: classe con EncodeFrame(), Configure(), FrameLength()
  - SbcEncoder.cpp: costruttore, _BuildHeader(), CRC-8
  - Test: istanziazione, header generation, CRC calcolo

Task A.2: Analisi subband (filter bank)
  - Matrice coseno di analisi (inversa della sintesi del decoder)
  - Windowing con coefficienti proto (sProto8/sProto4)
  - Test: segnale DC → tutte le subband a zero tranne sb0
  - Test: tono sinusoidale → energia concentrata nella subband corretta

Task A.3: Quantizzazione e bit allocation
  - Scale factor computation
  - SNR e Loudness allocation (identica al decoder)
  - Quantizzazione dei campioni subband
  - Test: bit allocation per bitpool noti, confronto con decoder

Task A.4: Joint stereo e packing
  - Joint stereo encoding (M/S decision per subband)
  - BitWriter per impacchettare i campioni quantizzati
  - Test: encode silenzio → decode → silenzio
  - Test: encode onda sinusoidale → decode → confronto SNR > 40dB

Task A.5: Roundtrip test completo
  - Encode → Decode ciclo con vari parametri
  - 44.1kHz stereo, 48kHz mono, 16kHz mono
  - Bitpool 2, 32, 53
  - Test: SNR > 30dB per tutti i casi
  - Integrazione in Jamfile (libbluetooth.so + bt_sbc_enc_test)


========================================================================
FASE B — A2DP Source (~500 righe)
========================================================================

Controparte di A2dpSink.cpp (552 righe). Invia audio codificato SBC
a cuffie remote usando AVDTP.

Task B.1: Classe A2dpSource base
  - A2dpSource.h: Connect(), Disconnect(), StartStream(), StopStream()
  - Riuso di AvdtpSession per AVDTP signaling
  - Test: compilazione, istanziazione

Task B.2: Endpoint discovery e codec negotiation
  - Discover → trova AVDTP_SEP_SINK sul dispositivo remoto
  - SetConfiguration con SBC (44.1kHz, Joint Stereo, bitpool 2-53)
  - Test: unit test con mock delle capabilities

Task B.3: Media transport e send loop
  - OpenMediaChannel() per canale media
  - SendAudio(const int16* pcm, size_t samples): encode → RTP → send
  - Pacing: rispetta timing basato su sample rate e dimensione frame
  - Test: formato pacchetto RTP corretto, media header SBC

Task B.4: Integrazione Jamfile e header pubblico
  - headers/os/bluetooth/A2dpSource.h
  - Aggiunta a libbluetooth.so Jamfile
  - bt_a2dp_source_test.cpp: test di integrazione

Task D.3: Controllo volume e parametri

Task E.3: Notifiche e SDP

Task E.4: Integrazione con media_server

FASE F — HFP Audio Gateway (~800 righe)
========================================================================

Ruolo AG: Haiku fa da "telefono" per le cuffie.
Le cuffie inviano comandi AT, Haiku risponde.

Task F.1: HfpAudioGateway classe base
  - HfpAudioGateway.h: Listen(), Accept(), SLC handshake
  - RFCOMM server socket (channel da SDP)
  - Test: compilazione, istanziazione

Task F.2: AT command responder
  - Riuso AtParser per parsing
  - Risposte AT+BRSF, AT+CIND, AT+CMER, AT+CHLD
  - Invio "+CIEV" per indicatori
  - Test: handshake SLC con comandi simulati

Task F.3: Audio routing SCO
  - Setup connessione SCO (vedi Fase G)
  - Routing audio bidirezionale
  - Test: setup SCO simulato

Task F.4: SDP record HFP AG
  - UUID 0x111F (HandsfreeAudioGateway)
  - Aggiunta a SdpServer
  - Test: SDP record presente

Task G.3: Kernel SCO module

========================================================================
DIPENDENZE TRA FASI
========================================================================

  A (SbcEncoder) ─┐
                   ├──► B (A2dpSource) ──► D (media add-on)
  C (SDP Source) ──┘                          │
                                              │
  E (AVRCP) ──────────────────────────────────┘

  F (HFP AG) ──────► G (SCO audio)

Fasi A, C, E sono indipendenti e possono procedere in parallelo.
Fase B richiede A.
Fase D richiede B.
Fase G richiede F.


========================================================================
PRIORITA'
========================================================================

1. A + B + C + D = audio musicale verso cuffie (MVP)
2. E = controlli cuffie (play/pause/volume)
3. F + G = chiamate voce (nice-to-have)


========================================================================
HARDWARE RICHIESTO
========================================================================

- Adattatore BT 2.1+EDR o superiore (BCM2070 sufficiente per A2DP)
- Cuffie BT con supporto A2DP Sink (qualsiasi cuffia standard)
- Per SCO (Fase G): potrebbe servire adattatore con endpoint isocroni
